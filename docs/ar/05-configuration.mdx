يتم التحكم في جميع سلوكيات خط الأنابيب من خلال ملفي YAML في `config/`.

## `config/validation.yaml`

### إعدادات واجهة برمجة التطبيقات

```yaml
api:
  base_url: "https://example-tenant.console.ves.volterra.io"
  tenant: "example-tenant"
  namespace: "example-namespace"
  timeout: 30
  retries: 3
  retry_delay: 2
```

| الحقل | الوصف |
|-------|-------------|
| `base_url` | عنوان URL الأساسي لواجهة برمجة تطبيقات وحدة تحكم F5 XC. يمكن تجاوزه بمتغير البيئة `F5XC_API_URL`. |
| `tenant` | اسم المستأجر المستخدم في مسارات واجهة برمجة التطبيقات. |
| `namespace` | مساحة الاسم الافتراضية لعمليات CRUD. |
| `timeout` | مهلة طلب HTTP بالثواني. |
| `retries` | عدد محاولات إعادة المحاولة عند الفشل. |
| `retry_delay` | عدد الثواني بين محاولات إعادة المحاولة. |

:::caution
قيم `base_url` و `tenant` و `namespace` في `validation.yaml` هي عناصر نائبة عامة. **يجب** عليك تكوين هذه القيم لبيئتك عبر متغيرات البيئة قبل تشغيل خط الأنابيب:

```bash
export F5XC_API_URL="https://example-tenant.console.ves.volterra.io"
export F5XC_TENANT="example-tenant"
export F5XC_NAMESPACE="example-namespace"
export F5XC_API_TOKEN="your-api-token"
```

تأخذ متغيرات البيئة الأسبقية على القيم الافتراضية في YAML أثناء التشغيل.
:::

### إعدادات التنزيل

```yaml
download:
  url: "https://docs.cloud.f5.com/docs-v2/downloads/f5-distributed-cloud-open-api.zip"
  output_dir: "specs/original"
  etag_cache: ".etag_cache"
```

| الحقل | الوصف |
|-------|-------------|
| `url` | عنوان URL لحزمة مواصفات OpenAPI الرسمية لـ F5 XC (ملف ZIP). |
| `output_dir` | مكان تخزين المواصفات المستخرجة. |
| `etag_cache` | الملف الذي يخزن علامة HTTP ETag للتنزيلات المشروطة. |

### فئات التحقق

عشر فئات تتوافق مع أنواع قيود مخطط OpenAPI. يمكن تمكين/تعطيل كل منها بشكل مستقل.

```yaml
validation_categories:
  string_length:
    enabled: true
    keywords: ["minLength", "maxLength"]
    description: "Validate string length boundaries"
```

| الفئة | الكلمات المفتاحية |
|----------|----------|
| `string_length` | `minLength`, `maxLength` |
| `pattern` | `pattern` |
| `numeric_bounds` | `minimum`, `maximum`, `exclusiveMinimum`, `exclusiveMaximum` |
| `required_fields` | `required` |
| `enum_values` | `enum` |
| `array_bounds` | `minItems`, `maxItems`, `uniqueItems` |
| `object_structure` | `additionalProperties`, `properties`, `propertyNames` |
| `composition` | `oneOf`, `anyOf`, `allOf` |
| `dependencies` | `dependentRequired`, `dependentSchemas` |
| `data_types` | `type`, `format` |

### إعدادات Schemathesis

```yaml
schemathesis:
  enabled: true
  max_examples: 100
  hypothesis_phases:
    - generate
    - target
  stateful_testing: true
  base_url_override: null
  auth_header: "Authorization"
  auth_prefix: "APIToken"
```

| الحقل | الوصف |
|-------|-------------|
| `enabled` | تبديل اختبار Schemathesis. |
| `max_examples` | الحد الأقصى لعدد حالات الاختبار المُنشأة لكل عملية. |
| `hypothesis_phases` | مراحل Hypothesis المراد تشغيلها (`generate`, `target`). |
| `stateful_testing` | تمكين الاختبار ذي الحالة القائم على الروابط عبر العمليات. |
| `base_url_override` | تجاوز عنوان URL الأساسي لواجهة برمجة التطبيقات للاختبارات. `null` يستخدم `api.base_url`. |
| `auth_header` | اسم ترويسة HTTP للمصادقة. |
| `auth_prefix` | تنسيق بادئة الرمز المميز (تُرسل الطلبات بصيغة `APIToken <token>`). |

### إعدادات Spectral

```yaml
spectral:
  enabled: true
  ruleset: "spectral-pipeline.yaml"
  auto_fix:
    oas3-api-servers: true
    info-contact: true
    operation-tags: true
    oas3-unused-component: true
    operation-operationId-unique: true
    oas3-valid-schema-example: true
    no-script-tags-in-markdown: true
  gate:
    max_errors: null
    max_warnings: null
  contact:
    name: "F5 Distributed Cloud"
    url: "https://docs.cloud.f5.com"
    email: "support@f5.com"
  servers:
    - url: "https://{tenant}.console.ves.volterra.io"
      description: "F5 Distributed Cloud API"
      variables:
        tenant:
          default: "example-tenant"
          description: "Your F5 XC tenant name"
  security_scheme:
    type: "apiKey"
    in: "header"
    name: "Authorization"
    description: "F5 XC API Token (format: APIToken <token>)"
```

| الحقل | الوصف |
|-------|-------------|
| `enabled` | تبديل فحص Spectral بالكامل. |
| `ruleset` | ملف تكوين Spectral المراد استخدامه (يستخدم خط الأنابيب `spectral-pipeline.yaml`). |
| `auto_fix` | خريطة من اسم القاعدة إلى قيمة منطقية. `true` تعني أن المُوفِّق سيصلح المخالفات. |
| `gate.max_errors` | الحد الأقصى المسموح به من الأخطاء في بوابة ما بعد التوفيق. `null` يعطل الفحص. |
| `gate.max_warnings` | الحد الأقصى المسموح به من التحذيرات. `null` يعطل الفحص. |
| `contact` | معلومات الاتصال المحقونة في `info.contact` بواسطة مُصلح `info-contact`. |
| `servers` | مصفوفة الخوادم المحقونة بواسطة مُصلح `oas3-api-servers`. |
| `security_scheme` | تعريف مخطط الأمان المحقون في كل مواصفة. |

## `config/endpoints.yaml`

يحدد نقاط النهاية الأساسية المستخدمة للتحقق المباشر من واجهة برمجة التطبيقات. يربط كل إدخال اسم مورد منطقي بملف المواصفة الخاص به ومجموعة واجهة برمجة التطبيقات ومسارات CRUD.

### بنية نقطة النهاية

```yaml
endpoints:
  healthcheck:
    resource: healthchecks
    domain_file: docs-cloud-f5-com.0124.public.ves.io.schema.healthcheck.ves-swagger.json
    api_group: config
    crud_operations:
      create: POST /api/config/namespaces/{namespace}/healthchecks
      read: GET /api/config/namespaces/{namespace}/healthchecks/{name}
      list: GET /api/config/namespaces/{namespace}/healthchecks
      update: PUT /api/config/namespaces/{namespace}/healthchecks/{name}
      delete: DELETE /api/config/namespaces/{namespace}/healthchecks/{name}
    test_priority: high
    description: "Health check configurations for origin monitoring"
```

| الحقل | الوصف |
|-------|-------------|
| `resource` | اسم مورد واجهة برمجة التطبيقات (يُستخدم في مسارات URL). |
| `domain_file` | اسم ملف مواصفة OpenAPI في `specs/original/`. |
| `api_group` | بادئة مجموعة واجهة برمجة التطبيقات (`config`, `web`, إلخ). |
| `crud_operations` | طريقة HTTP والمسار لكل عملية CRUD. |
| `test_priority` | `high` أو `medium` أو `low` -- يتحكم في ترتيب تنفيذ الاختبارات. |
| `description` | وصف مقروء للمورد. |

### نقاط النهاية الأساسية الحالية

10 نقاط نهاية عبر ثلاثة مجالات:

| نقطة النهاية | المجال | الأولوية |
|----------|--------|----------|
| `healthcheck` | Virtual | عالية |
| `origin_pool` | Virtual | عالية |
| `app_firewall` | Virtual | عالية |
| `service_policy` | Virtual | متوسطة |
| `api_definition` | API Security | عالية |
| `api_discovery` | API Security | متوسطة |
| `api_groups` | API Security | متوسطة |
| `code_base_integration` | API Security | منخفضة |
| `data_type` | Data Privacy | متوسطة |
| `sensitive_data_policy` | Data Privacy | متوسطة |

### إضافة نقاط نهاية جديدة

لإضافة نقطة نهاية جديدة للتحقق:

1. ابحث عن اسم ملف المواصفة في `specs/original/` (الصيغة: `docs-cloud-f5-com.NNNN.*.ves-swagger.json`)
2. أضف إدخالاً تحت `endpoints:` متبعاً البنية أعلاه
3. أضف ربط ملف المجال تحت `domain_files:` مع رقم أولوية
4. أضف اسم نقطة النهاية إلى `test_order:` في الموضع المطلوب

:::tip
يتحقق إطار العمل من جميع المواصفات البالغ عددها 268 أثناء التوفيق بغض النظر عما هو مدرج في `endpoints.yaml`. يتحكم تكوين نقاط النهاية في الموارد التي تخضع لاختبار واجهة برمجة التطبيقات المباشر باستخدام Schemathesis.
:::
