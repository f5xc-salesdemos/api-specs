يستخدم المشروع [Spectral](https://stoplight.io/open-source/spectral) لفحص OAS3 ببنية ثنائية التكوين تفصل أدوات CI/الأدوات الخارجية عن المنطق المخصص لخط الأنابيب.

## بنية التكوين الثنائية

### `.spectral.yaml` -- التكوين الأساسي

يُستخدم بواسطة GitHub Super-Linter وأي أداة CI خارجية. يمتد من `spectral:oas` (مجموعة قواعد OAS المدمجة). لا يحتوي على دوال مخصصة -- مما يبقيه متوافقًا مع الأدوات التي تشغّل Spectral دون الوصول إلى دليل `spectral/functions/`.

```yaml
extends:
  - "spectral:oas"

rules:
  path-params: "off"
  operation-tags: warn
  oas3-api-servers: warn
  info-contact: warn
  oas3-unused-component: warn
  operation-operationId-unique: error
  oas3-valid-schema-example: error
  no-script-tags-in-markdown: warn
  operation-description: info
  info-description: info
```

### `spectral-pipeline.yaml` -- تكوين خط الأنابيب

يُستخدم بواسطة `scripts/spectral_lint.py` (في وضعي الاكتشاف والبوابة). يمتد من التكوين الأساسي ويضيف الدالة المخصصة `f5-path-params`:

```yaml
extends:
  - "./.spectral.yaml"

functionsDir: "./spectral/functions"
functions:
  - f5-path-params

rules:
  f5-path-params:
    description: "Path parameters must be declared and used"
    message: "{{error}}"
    severity: error
    given: "$.paths[*]"
    then:
      function: f5-path-params
```

## لماذا تم تعطيل `path-params`

تُشير قاعدة `path-params` المدمجة في Spectral بشكل خاطئ إلى واجهات برمجة تطبيقات F5 XC التي تستخدم أسماء معاملات منقوطة مثل `{metadata.namespace}` و `{metadata.name}`. القاعدة المدمجة لا تحلل النقاط في أسماء المعاملات، لذلك تُبلغ عنها كمعاملات غير معرّفة.

الدالة المخصصة `f5-path-params` (`spectral/functions/f5-path-params.js`) تتعامل بشكل صحيح مع الأسماء المنقوطة عن طريق مطابقة محتوى `{placeholder}` الكامل مع أسماء المعاملات المعرّفة. تتحقق من المعاملات على مستوى المسار ومستوى العملية، وتُبلغ عن الأخطاء في كلا الاتجاهين:

- عنصر نائب في مسار URL لا يوجد له تعريف معامل مطابق
- معامل مسار معرّف لا يظهر في مسار URL

## مرجع القواعد

### القواعد القابلة للإصلاح التلقائي

يتم إصلاح هذه القواعد تلقائيًا بواسطة المُوفِّق عند اكتشاف المخالفات. انظر [الإصلاحات المطبقة](/03-fixes-applied/) للحصول على تفاصيل حول كل إصلاح.

| القاعدة | الخطورة | طريقة الإصلاح | الوصف |
|------|----------|------------|-------------|
| `oas3-api-servers` | warn | `_add_servers` | يجب أن تحتوي المواصفة على كتلة `servers` |
| `info-contact` | warn | `_add_contact` | يجب أن يتضمن `info` معلومات `contact` |
| `operation-tags` | warn | `_add_tags` | يجب أن تحتوي كل عملية على وسم واحد على الأقل |
| `oas3-unused-component` | warn | `_remove_unused_component` | لا مخططات مكونات غير مرجعية |
| `operation-operationId-unique` | error | `_deduplicate_operation_id` | يجب أن تكون جميع معرّفات العمليات فريدة |
| `oas3-valid-schema-example` | error | `_fix_schema_example` | يجب أن تتطابق الأمثلة/القيم الافتراضية مع مخططاتها |
| `no-script-tags-in-markdown` | warn | `_strip_script_tags` | لا وسوم `<script>` في الأوصاف |
| `f5-path-params` | error | -- | يجب أن تكون معاملات المسار معرّفة ومستخدمة |

:::note
`f5-path-params` هي قاعدة بخطورة error لكنها لا تُصلح تلقائيًا. تشير المخالفات إلى عيب حقيقي في المواصفة يتطلب مراجعة يدوية.
:::

### القواعد غير القابلة للإصلاح

تم تخفيض خطورة هذه القواعد إلى `info` لأنه لا يمكن إصلاحها تلقائيًا بشكل ذي معنى:

| القاعدة | الخطورة | الوصف |
|------|----------|-------------|
| `operation-description` | info | يجب أن تحتوي العمليات على وصف |
| `info-description` | info | يجب أن تحتوي معلومات API على وصف |

## التكامل مع خط الأنابيب

يعمل محول Spectral (`scripts/spectral_lint.py`) في وضعين:

### وضع الاكتشاف (قبل التوفيق)

```bash
make spectral-lint
# Equivalent to: python -m scripts.spectral_lint --mode discover
```

يفحص `specs/original/` ويكتب `reports/spectral_report.json`. يستهلك المُوفِّق هذا التقرير إلى جانب تقرير التحقق.

### وضع البوابة (بعد التوفيق)

```bash
make spectral-gate
# Equivalent to: python -m scripts.spectral_lint --mode gate
```

يفحص `release/specs/` ويكتب `reports/spectral_gate_report.json`. يُرجع رمز خروج 1 إذا تجاوزت المخالفات الحدود المكوّنة.

### تعيين المخالفات

تُحوَّل مخالفات Spectral إلى كائنات `Discrepancy` بثلاثة أنواع فرعية:

| فئة قاعدة Spectral | DiscrepancyType | أمثلة على القواعد |
|------------------------|-----------------|---------------|
| عنصر مطلوب مفقود | `SPECTRAL_MISSING` | `oas3-api-servers`, `info-contact`, `operation-tags` |
| عنصر موجود غير صالح | `SPECTRAL_INVALID` | `oas3-valid-schema-example`, `no-script-tags-in-markdown` |
| كود ميت | `SPECTRAL_UNUSED` | `oas3-unused-component` |
